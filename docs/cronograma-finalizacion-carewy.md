# Cronograma de finalizacion de CareWy

## Objetivo

Completar el porcentaje restante del proyecto hasta una version final mas robusta, priorizando estabilidad, automatizacion, validacion inteligente y despliegue.

## Funcionalidades pendientes priorizadas

### Prioridad alta

- Integrar OCR e IA real para validar comprobantes.
- Integrar IA real para validar documentos del cuidador.
- Completar chat en tiempo real.
- Preparar despliegue estable de frontend, backend y base de datos.

### Prioridad media

- Implementar push notifications.
- Agregar pruebas automatizadas de flujos criticos.
- Reforzar seguridad y control de errores.

### Prioridad baja

- Pulido visual adicional.
- Mejoras de rendimiento y division de bundles.
- Ajustes de accesibilidad y experiencia avanzada.

## Cronograma propuesto

### Semana 1

Objetivo: cerrar validacion documental y de pagos.

- Integrar OCR para extraer nombre, monto y motivo del comprobante.
- Comparar datos del comprobante con la reserva.
- Integrar validacion automatizada para expediente del cuidador.
- Probar flujo admin de aprobaciones y rechazos.

Entregable:

- Modulo de revision automatica inicial funcionando.

### Semana 2

Objetivo: mejorar comunicacion y experiencia operativa.

- Implementar chat realmente en tiempo real.
- Mejorar notificaciones internas.
- Definir estructura para push notifications.

Entregable:

- Mensajeria mas inmediata y mejor experiencia de seguimiento.

### Semana 3

Objetivo: calidad y seguridad.

- Agregar pruebas de autenticacion, reservas y comprobantes.
- Validar carga de archivos y restricciones.
- Endurecer manejo de sesiones e intentos fallidos.

Entregable:

- Base tecnica mas estable y segura.

### Semana 4

Objetivo: despliegue y cierre.

- Publicar frontend.
- Publicar backend.
- Configurar base de datos final y variables de entorno.
- Probar flujo completo extremo a extremo en entorno final.

Entregable:

- Version desplegada y lista para cierre del proyecto.

## Resumen corto para defender en clase

Puedes decirlo asi:

> El sistema ya cumple el flujo principal con persistencia real. Lo que falta se concentra en integraciones avanzadas de IA, tiempo real, pruebas automatizadas y despliegue final. El cronograma esta organizado en cuatro semanas para cerrar primero validacion inteligente, luego mensajeria en tiempo real, despues calidad tecnica y finalmente publicacion del sistema.
